Variables and Constants

Variable Declaration

Variables introduce names that can later be assigned new values. In Ego you declare a new variable using the := operator.

declare.ego
example
count    := 0
message  := "hello"
flag     := true
:= operator. Using := both declares a new variable and assigns its initial value in one step.
Scope rules. A variable is visible from the point where it is declared until the end of the current block. Inner blocks can see variables declared in outer blocks.
Shadowing. Declaring a new variable with the same name in an inner block creates a new binding that temporarily shadows the outer one. When the block ends, the outer variable is visible again.

Assignment

After a variable is declared, you update its value with the = operator.

assign.ego
example
count := 1
count = count + 1

name := "Ego"
name = name + " Lang"
= operator. The plain = operator only assigns to an existing variable; it does not declare new ones.
Valid left-hand sides. The left-hand side of = must be a writable location, typically a variable name. Assigning to literals or temporary expressions is not allowed.
Runtime errors on undeclared identifiers. If you use = on a name that has not been declared with :=, Ego reports a runtime error instead of silently creating a new variable.

Constants

Constants are bindings that cannot be reassigned after they are created. They are useful for configuration values and named literals.

const.ego
example
$PI    := 3.14
$greet := "hello"

# later...
$PI = 3.14159   # error: cannot assign to constant
Constant declaration syntax. Use the $ symbol followed by a name, :=, and an expression to declare a constant. It does not have to be uppercase.
Immutability rules. Once a constant is defined, you cannot assign a new value to it. Attempting to do so results in a runtime error.
What values can be constant. Any value that can appear on the right-hand side of = can be used in a constant declaration, including numbers, strings, booleans, and more complex structures.
Runtime behavior on reassignment. Reassigning a constant is an error at runtime; Ego stops execution and reports what went wrong.
