Expressions and Operators

Arithmetic Operators

Ego supports the usual arithmetic operators on numbers: +, -, *, /, and %.

arith.ego
example
a := 10
b := 3

sum  := a + b
sub  := a - b
prod := a * b
quot := a / b
mod  := a % b
Integer vs float behavior. When both operands are integers, / performs integer division. If either side is a float, arithmetic produces a float result. The % operator only works on integers.

Comparison Operators

Comparisons produce boolean results and can be combined in conditions.

compare.ego
example
x := 10
y := 20

put(x == y)  # false
put(x != y)  # true
put(x < y)   # true
put(x <= 10) # true
put(y > x)   # true
put(y >= 20) # true
Operators. Ego provides ==, !=, <, >, <=, and >=.

Logical and Prefix Operators

Logical operators combine boolean expressions, and prefix operators let you negate booleans or numbers.

logic.ego
example
a := true
b := false

put(a and b) # false
put(a or b)  # true

# prefix operators
put(!a)      # false
put(!b)      # true

x := 5
y := -x     # numeric negation
Logical operators. Use and and or to combine boolean expressions. They work with the usual truth tables.
Prefix !. The ! operator negates a boolean: !true is false, and vice versa.
Prefix -. The unary - operator negates a numeric value, turning positive numbers into negative ones and vice versa.
Precedence with logical operators. The ! prefix binds more tightly than and, which in turn binds more tightly than or.

Operator Precedence

Operator precedence determines which parts of an expression are evaluated first. Parentheses can always be used to make the intended order explicit.

precedence.ego
example
x := 1 + 2 * 3      # 1 + (2 * 3)
y := (1 + 2) * 3    # grouped explicitly

ok := !false and 1 < 2 or false
Precedence table (high to low). -x, !x → *, /, % → +, - → comparisons (==, !=, <, >, <=, >=) → and → or.
Associativity. Most binary operators are left-associative, so a - b - c is interpreted as (a - b) - c.
Disallowed expressions. Chained assignments like a = b = 1 are not allowed; the left-hand side of = must be a single assignable location.
