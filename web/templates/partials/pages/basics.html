Language Basics

Program Structure

An Ego program is just a sequence of statements executed from top to bottom. There is no special project format or module system required to get started.

Statements - Each line normally contains a single statement: an assignment, expression, or control-flow construct (like an if or a for loop).
Newlines - A newline ends a statement or expression. The ; is only used to close blocks. They do not go at the end of every line.
Block termination with ; - Multi-line constructs (such as conditionals, loops or function declarations) are closed by a final statement that ends with ; to mark the end of the block.
Top-level execution - Currently, code at the top level of a file runs immediately when the file is executed. There is no required entrypoint function for scripts. However, in the future Ego may require a main function entrypoint

Comments

Comments are ignored by the interpreter and are only for humans. Use them to explain intent, assumptions, or non-obvious behavior.

comments.ego
example
# this is a line comment
x := 10  # comments can follow code on the same line

# the interpreter ignores everything after '#'
put(x)
Line comments. Ego uses # for line comments. Everything from # to the end of the line is ignored.
Comment rules. You can place comments on their own line or after code on the same line. Comments cannot appear in the middle of a token (for example, inside a number or identifier).
Nesting. Only line comments are supported, so comments do not nest. Another # inside a comment has no special meaning.

Identifiers

Identifiers are the names you give to variables, functions, and other user-defined symbols.

identifiers.ego
example
name     := "Ego"
answer42 := 42
_is_temp := true
$PI      := 3.14

put(name + ' is ' + str(answer42))
Valid characters. Identifiers start with a letter a-z or A-Z, the $ symbol, or an underscore _, and may contain letters, digits, and underscores after that.
Case sensitivity. Identifiers are case-sensitive, so value and Value are different names.
Reserved words. Certain short words are reserved for language syntax (for example, if, else, for, nil, ret). These cannot be used as identifiers.
Special identifiers. The language reserves names like $0, $1, $2, and so on for special purposes (such as positional parameters passed to a script). Avoid using these for your own variables.
